# 微信小程序登录调试指南

## 登录流程说明

### 1. 用户点击"点击登录"按钮
↓
### 2. 调用 `onGetUserInfo()` 函数
↓
### 3. 显示"登录中..."加载框
↓
### 4. 调用 `app.wxLogin()` 
↓
### 5. 执行 `wx.login()` 获取code
↓
### 6. 发送POST请求到后端 `/api/miniapp/user/login`
↓
### 7. 后端返回用户信息和token
↓
### 8. 保存token和userId到本地存储
↓
### 9. 显示登录成功提示
↓
### 10. 页面显示用户信息

## 调试步骤

### 第一步：打开微信开发者工具的调试窗口

1. 在微信开发者工具中，点击右上角的 **调试器** 或按 `F12`
2. 选择 **Console** 标签页
3. 这样你可以看到所有的日志输出

### 第二步：点击登录按钮并观察日志

1. 在小程序中点击"我的"页面
2. 点击"点击登录"按钮
3. 在Console中观察输出的日志信息

**你应该看到类似这样的日志：**
```
开始登录...
wx.login success: {code: "xxx", errMsg: "login:ok"}
login request success: {data: {code: 200, data: {userId: 1, ...}}}
登录成功: {userId: 1, openid: "oTest_xxx", ...}
```

### 第三步：如果看到错误日志

**常见错误1：网络请求失败**
```
login request fail: {errMsg: "request:fail"}
```
**解决方案：**
- 检查后端服务是否运行在 http://localhost:8080
- 检查微信开发者工具的"详情"中是否勾选了"不校验合法域名..."

**常见错误2：后端返回错误**
```
login request success: {data: {code: 500, message: "...错误信息..."}}
```
**解决方案：**
- 查看错误信息
- 检查后端服务的日志

**常见错误3：没有任何日志输出**
```
（什么都没有）
```
**解决方案：**
- 检查按钮是否真的被点击了
- 检查 `onGetUserInfo` 函数是否正确绑定到按钮

## 手动测试后端登录接口

### 使用浏览器测试

1. 打开浏览器
2. 访问：`http://localhost:8080/api/miniapp/user/login`
3. 这会显示一个错误（因为需要POST请求）

### 使用Postman测试

1. 打开Postman
2. 创建新请求
3. 方法：`POST`
4. URL：`http://localhost:8080/api/miniapp/user/login`
5. Body（JSON）：
```json
{
  "code": "test_code"
}
```
6. 点击 **Send**
7. 应该看到返回的JSON数据

**成功响应示例：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "userId": 1,
    "openid": "oTest_1234567890",
    "nickname": "微信用户",
    "token": "mock-token-1"
  }
}
```

## 验证数据库

### 检查是否创建了新用户

在SQLyog中执行：
```sql
SELECT * FROM user ORDER BY create_time DESC LIMIT 5;
```

应该能看到新创建的用户记录。
