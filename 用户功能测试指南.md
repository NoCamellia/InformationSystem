## 问题1：微信小程序登录功能

### 后端实现说明

我已经为你创建了完整的登录功能：

#### 1. 登录流程
```
小程序端 → 调用 wx.login() 获取 code
    ↓
发送 code 到后端 /api/miniapp/user/login
    ↓
后端调用微信API验证 code，获取 openid
    ↓
查询数据库是否存在该用户
    ↓
不存在则创建新用户
    ↓
返回用户信息和 token
    ↓
小程序保存 token 到本地存储
```

#### 2. 后端代码位置
- **Controller**: `server/src/main/java/com/info/consultation/controller/miniapp/MiniUserController.java`
- **Service**: `server/src/main/java/com/info/consultation/service/impl/UserServiceImpl.java`

#### 3. 登录API接口
```
POST /api/miniapp/user/login
请求体: { "code": "微信返回的code" }
返回: { 
  "code": 200,
  "data": {
    "userId": 1,
    "openid": "oTest_xxx",
    "nickname": "微信用户",
    "token": "mock-token-1"
  }
}
```

## 问题2：用户收藏和浏览历史记录

### 后端实现说明

我已经为你创建了完整的收藏和历史记录功能：

#### 1. 收藏功能

**数据库表**: `user_collect`
```sql
CREATE TABLE user_collect (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  article_id BIGINT NOT NULL,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_user_article (user_id, article_id)
);
```

**后端代码位置**:
- **Controller**: `server/src/main/java/com/info/consultation/controller/miniapp/MiniCollectController.java`
- **Service**: `server/src/main/java/com/info/consultation/service/impl/UserCollectServiceImpl.java`
- **Entity**: `server/src/main/java/com/info/consultation/entity/UserCollect.java`

**API接口**:
```
# 获取用户收藏列表
GET /api/miniapp/collect/list?userId=1
返回: [{ id, userId, articleId, createTime }, ...]

# 检查是否已收藏
GET /api/miniapp/collect/check/1/1
返回: { "code": 200, "data": true/false }

# 添加收藏
POST /api/miniapp/collect/add?userId=1&articleId=1
返回: { "code": 200, "message": "操作成功" }

# 取消收藏
DELETE /api/miniapp/collect/remove?userId=1&articleId=1
返回: { "code": 200, "message": "操作成功" }
```

#### 2. 浏览历史功能

**数据库表**: `user_history`
```sql
CREATE TABLE user_history (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  article_id BIGINT NOT NULL,
  view_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_user_id (user_id),
```

**后端代码位置**:

#### 步骤4：在小程序中集成

**在文章详情页面添加收藏功能**:

```javascript
// pages/detail/detail.js
onCollectTap() {
  const { article, isCollected } = this.data
  const userId = wx.getStorageSync('userId') || 1 // 从本地存储获取用户ID
  
  if (isCollected) {
    wx.showToast({
      title: '已经收藏过了',
      icon: 'none'
    })
    return
  }

  api.addCollect(userId, article.id).then(() => {
    this.setData({
      isCollected: true,
      'article.collectCount': article.collectCount + 1
    })
    wx.showToast({
      title: '收藏成功',
      icon: 'success'
    })
  }).catch(err => {
    console.error('收藏失败', err)
  })
}
```

**在我的页面显示收藏列表**:

```javascript
// pages/my/my.js
onCollectTap() {
  if (!this.data.hasUserInfo) {
    wx.showToast({
      title: '请先登录',
      icon: 'none'
    })
    return
  }
  
  const userId = wx.getStorageSync('userId')
  api.getCollectList(userId).then(res => {
    wx.navigateTo({
      url: '/pages/collect/collect?data=' + JSON.stringify(res)
    })
  })
}
```

## 完整的数据流向

### 收藏流程
```
用户点击收藏按钮
    ↓
小程序调用 api.addCollect(userId, articleId)
    ↓
发送 POST 请求到 /api/miniapp/collect/add
    ↓
后端 UserCollectService.addCollect() 方法
    ↓
检查是否已收藏（查询 user_collect 表）
    ↓
如果未收藏，插入新记录到 user_collect 表
    ↓
返回成功响应
    ↓
小程序更新UI，显示"已收藏"
    ↓
用户可以在"我的收藏"页面查看
```

### 浏览历史流程
```
用户打开文章详情
    ↓
小程序调用 api.addHistory(userId, articleId)
    ↓
发送 POST 请求到 /api/miniapp/history/add
    ↓
后端 UserHistoryService.addHistory() 方法
    ↓
插入新记录到 user_history 表（记录浏览时间）
    ↓
返回成功响应
    ↓
用户可以在"浏览历史"页面查看最近浏览的文章
```

## 常见问题

**Q: 为什么登录后还是看不到用户信息？**
A: 需要确保：
1. 后端服务正常运行
2. 小程序正确调用了登录API
3. 本地存储正确保存了用户ID

**Q: 收藏数据没有保存到数据库？**
A: 检查：
1. 后端是否正常运行
2. 是否正确传递了userId和articleId
3. 数据库连接是否正常

**Q: 如何查看所有用户的收藏？**
A: 在SQLyog中执行：
```sql
SELECT u.nickname, a.title, uc.create_time 
FROM user_collect uc
JOIN user u ON uc.user_id = u.id
JOIN article a ON uc.article_id = a.id
ORDER BY uc.create_time DESC;
```
