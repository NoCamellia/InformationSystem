# 收藏和浏览历史功能优化说明

## 🔧 已修复的问题

### 问题1：收藏和历史页面显示白框
**原因**：API返回的是数据库记录（UserCollect、UserHistory），而不是文章对象（Article）

**解决方案**：
- 后端API现在返回完整的文章对象
- 前端可以直接显示文章信息

### 问题2：同一篇文章浏览历史重复增加
**原因**：每次打开文章都会创建新的浏览记录

**解决方案**：
- 检查是否已存在该文章的浏览记录
- 如果存在，更新浏览时间
- 如果不存在，创建新记录
- 这样同一篇文章只会在历史中出现一次

### 2. MiniHistoryController 修改
```java
// 修改前：返回 List<UserHistory>
public Result<List<UserHistory>> getHistoryList(...)

// 修改后：返回 List<Article>（去重）
public Result<List<Article>> getHistoryList(...) {
    List<UserHistory> histories = userHistoryService.getUserHistory(userId, limit);
    // 获取浏览历史的文章详情，并去重
    List<Article> articles = histories.stream()
        .map(history -> articleService.getById(history.getArticleId()))
        .distinct()
        .collect(Collectors.toList());
    return Result.success(articles);
}
```

### 3. UserHistoryService 修改
```java
// 修改前：每次都创建新记录
public boolean addHistory(Long userId, Long articleId) {
    UserHistory history = new UserHistory();
    history.setUserId(userId);
    history.setArticleId(articleId);
    history.setViewTime(LocalDateTime.now());
    return this.save(history);
}

// 修改后：检查是否存在，存在则更新，不存在则创建
public boolean addHistory(Long userId, Long articleId) {
    LambdaQueryWrapper<UserHistory> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(UserHistory::getUserId, userId)
           .eq(UserHistory::getArticleId, articleId);
    UserHistory existingHistory = this.getOne(wrapper);
    
    if (existingHistory != null) {
        // 如果已存在，更新浏览时间
        existingHistory.setViewTime(LocalDateTime.now());
        return this.updateById(existingHistory);
    } else {
        // 如果不存在，创建新记录
        UserHistory history = new UserHistory();
        history.setUserId(userId);
        history.setArticleId(articleId);
        history.setViewTime(LocalDateTime.now());
        return this.save(history);
    }
}
```

### 查看最新的浏览时间
```sql
SELECT uh.id, u.nickname, a.title, uh.view_time
FROM user_history uh
JOIN user u ON uh.user_id = u.id
JOIN article a ON uh.article_id = a.id
ORDER BY uh.view_time DESC
LIMIT 10;
```

## 📊 现在的数据流向

### 收藏流程
```
用户点击收藏
    ↓
后端保存到 user_collect 表
    ↓
用户进入"我的收藏"
    ↓
后端查询 user_collect 表
    ↓
根据 article_id 获取文章详情
    ↓

    ↓
前端显示文章列表
```

```
    ↓
存在：更新浏览时间
不存在：创建新记录
    ↓
用户进入"浏览历史"
    ↓
后端查询 user_history 表
    ↓
根据 article_id 获取文章详情
    ↓
去重（同一篇文章只显示一次）
    ↓
返回完整的 Article 对象列表
    ↓
前端显示文章列表
```
