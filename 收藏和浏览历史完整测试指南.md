## 🔍 在SQLyog中验证数据

### 查看收藏记录
```sql
-- 查看所有收藏
SELECT * FROM user_collect;

-- 查看某用户的收藏
SELECT * FROM user_collect WHERE user_id = 1;

-- 查看收藏的文章详情
SELECT uc.id, u.nickname, a.title, uc.create_time
FROM user_collect uc
JOIN user u ON uc.user_id = u.id
JOIN article a ON uc.article_id = a.id
ORDER BY uc.create_time DESC;
```

### 查看浏览历史
```sql
-- 查看所有浏览历史
SELECT * FROM user_history;

-- 查看某用户的浏览历史
SELECT * FROM user_history WHERE user_id = 1 ORDER BY view_time DESC;

-- 查看浏览历史的文章详情
SELECT uh.id, u.nickname, a.title, uh.view_time
FROM user_history uh
JOIN user u ON uh.user_id = u.id
JOIN article a ON uh.article_id = a.id
ORDER BY uh.view_time DESC;
```

### 统计数据
```sql
-- 统计某用户的收藏数
SELECT COUNT(*) as collect_count FROM user_collect WHERE user_id = 1;

-- 统计某用户的浏览历史数
SELECT COUNT(*) as history_count FROM user_history WHERE user_id = 1;

-- 统计每个用户的收藏数
SELECT user_id, COUNT(*) as collect_count FROM user_collect GROUP BY user_id;
```

## 📊 数据流向图

### 收藏流程
```
用户点击收藏按钮
    ↓
detail.js 的 onCollectTap() 函数
    ↓
调用 api.addCollect(userId, articleId)
    ↓
发送 POST 请求到 /api/miniapp/collect/add
    ↓
后端 UserCollectService.addCollect() 方法
    ↓
检查是否已收藏
    ↓
插入新记录到 user_collect 表
    ↓
返回成功响应
    ↓
小程序显示"收藏成功"提示
    ↓
用户进入"我的收藏"页面
    ↓
调用 api.getCollectList(userId)
    ↓
后端返回该用户的所有收藏
    ↓
页面显示收藏的文章列表
```

### 浏览历史流程
```
用户打开文章详情
    ↓
detail.js 的 onLoad() 函数
    ↓
调用 addHistory(userId, articleId)
    ↓
调用 api.addHistory(userId, articleId)
    ↓
发送 POST 请求到 /api/miniapp/history/add
    ↓
后端 UserHistoryService.addHistory() 方法
    ↓
插入新记录到 user_history 表
    ↓
返回成功响应
    ↓
用户进入"浏览历史"页面
    ↓
调用 api.getHistoryList(userId, limit)
    ↓
后端返回该用户的浏览历史
    ↓
页面显示浏览过的文章列表
```

## 🎯 答辩时的说明

**老师问："收藏和浏览历史是怎么实现的？"**

你可以这样回答：
> "用户点击收藏按钮时，小程序会调用后端的收藏API，将用户ID和文章ID发送到服务器。后端会检查是否已收藏，如果没有则插入新记录到 `user_collect` 表。浏览历史是在用户打开文章详情时自动记录的，会插入到 `user_history` 表。用户可以在'我的收藏'和'浏览历史'页面查看这些数据，这些数据都是从数据库实时获取的。"
