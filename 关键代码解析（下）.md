## 7. 收藏服务 - 防重复

```java
@Override
public boolean addCollect(Long userId, Long articleId) {
    if (isCollected(userId, articleId)) {
        return false;
    }
    UserCollect collect = new UserCollect();
    collect.setUserId(userId);
    collect.setArticleId(articleId);
    return this.save(collect);
}

@Override
public boolean isCollected(Long userId, Long articleId) {
    LambdaQueryWrapper<UserCollect> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(UserCollect::getUserId, userId)
           .eq(UserCollect::getArticleId, articleId);
    return this.count(wrapper) > 0;
}
```

addCollect先调用isCollected防止重复，这是后端的业务防护。

**问题**：两次数据库查询能否优化？

回答：可以给(user_id, article_id)加联合唯一索引，直接插入并捕获唯一键冲突异常，只需一次查询。当前实现更直观，毕业设计数据量下够用。

## 8. 浏览历史 - 去重更新

```java
@Override
public boolean addHistory(Long userId, Long articleId) {
    LambdaQueryWrapper<UserHistory> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(UserHistory::getUserId, userId)
           .eq(UserHistory::getArticleId, articleId);
    UserHistory existing = this.getOne(wrapper);

    if (existing != null) {
        existing.setViewTime(LocalDateTime.now());
        return this.updateById(existing);
    } else {
        UserHistory history = new UserHistory();
        history.setUserId(userId);
        history.setArticleId(articleId);
        history.setViewTime(LocalDateTime.now());
        return this.save(history);
    }
}
```

同一篇文章不重复出现，只更新浏览时间，历史列表按viewTime倒序反映最近阅读。

**问题**：为什么不每次都新建记录？

回答：每次新建会导致历史列表重复，体验差。每篇文章只出现一次、记录最近浏览时间才是合理设计。

## 9. 小程序请求封装 api.js

```javascript
const request = (options) => {
  return new Promise((resolve, reject) => {
    wx.request({
      url: app.globalData.baseUrl + options.url,
      method: options.method || 'GET',
      header: {
        'content-type': 'application/json',
        'Authorization': 'Bearer ' + (app.globalData.token || '')
      },
      success: (res) => {
        if (res.data.code === 200) {
          resolve(res.data.data)
        } else {
          wx.showToast({ title: res.data.message || '请求失败', icon: 'none' })
          reject(res.data)
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络错误', icon: 'none' })
        reject(err)
      }
    })
  })
}

// 参数拼在URL中，因为后端用@RequestParam接收
const addCollect = (userId, articleId) => {
  return request({
    url: `/collect/add?userId=${userId}&articleId=${articleId}`,
    method: 'POST'
  })
}
```

封装成Promise，统一处理成功和失败，每个请求自动携带token。

**问题**：参数为什么拼在URL而不是请求体？

回答：这是开发中的真实踩坑。后端@RequestParam只从URL查询字符串取值，放POST请求体里取不到会返回400错误。解决方案是参数拼到URL，或后端改用@RequestBody接收。

## 10. 微信登录 app.js

```javascript
wxLogin() {
  return new Promise((resolve, reject) => {
    wx.login({
      success: (res) => {
        if (res.code) {
          wx.request({
            url: this.globalData.baseUrl + '/user/login',
            method: 'POST',
            data: { code: res.code },
            success: (result) => {
              if (result.data && result.data.code === 200) {
                const { token, userId } = result.data.data
                wx.setStorageSync('token', token)   // 持久化存储
                wx.setStorageSync('userId', userId)
                this.globalData.token = token       // 内存存储
                resolve(result.data.data)
              } else {
                reject(result.data?.message || '登录失败')
              }
            },
            fail: reject
          })
        }
      },
      fail: reject
    })
  })
}
```

token同时存在内存（globalData）和本地存储（localStorage），重开小程序不需要重新登录。

**问题**：微信code是什么，能直接识别用户吗？

回答：code是wx.login()返回的临时凭证，有效期5分钟，每次都不同，不能识别用户。后端用code调微信接口换取openid，openid才是用户在该小程序的固定唯一标识。

## 11. 收藏列表返回文章对象

```java
@GetMapping("/list")
public Result<List<Article>> getCollectList(@RequestParam Long userId) {
    List<UserCollect> collects = userCollectService.getUserCollects(userId);
    List<Article> articles = collects.stream()
        .map(collect -> articleService.getById(collect.getArticleId()))
        .collect(Collectors.toList());
    return Result.success(articles);
}
```

前端要显示文章标题、封面、摘要，所以后端需要根据articleId查出完整Article对象，而不是只返回user_collect表的记录。数据组装在后端完成，避免前端多次请求。

**问题**：收藏100篇文章就查询100次数据库，怎么优化？

回答：用IN查询一次取出所有文章，SELECT * FROM article WHERE id IN (1,2,3...)，对应MyBatis Plus的listByIds方法，把N次查询优化为1次。当前stream实现简单直观，数据量小时没问题。
